# Project Overview

This repo contains custom dev container Features published to GHCR (GitHub Container Registry). It follows the [devcontainers/feature-starter](https://github.com/devcontainers/feature-starter) structure.

## Repo Structure

- `src/<feature>/` - Feature source: `devcontainer-feature.json` (metadata + options) and `install.sh` (install script)
- `test/<feature>/` - Tests: `test.sh` (default test), `scenarios.json` + matching `.sh` files (scenario tests)
- `.github/workflows/test.yaml` - CI: tests features against multiple base images
- `.github/workflows/release.yaml` - Publishes features to GHCR on push to `master`

## Adding a Feature

1. Create `src/<feature-id>/devcontainer-feature.json` with `name`, `id`, `version`, `description`, `options`
2. Create `src/<feature-id>/install.sh` - options are available as uppercased env vars (e.g. `myOption` becomes `$MYOPTION`)
3. Create `test/<feature-id>/test.sh` for default tests (uses `dev-container-features-test-lib`)
4. Optionally add `test/<feature-id>/scenarios.json` with scenario tests
5. Add the feature to the `matrix.features` arrays in `.github/workflows/test.yaml`

## Testing

```bash
# Run default tests (uses default option values)
devcontainer features test -f <feature> --skip-scenarios .

# Run scenario tests
devcontainer features test -f <feature> --skip-autogenerated --skip-duplicated .
```

## Conventions

- Install scripts use `#!/bin/bash` with `set -e`
- Binaries are installed to `/usr/local/bin/`
- When downloading from GitHub releases, use the GitHub API to resolve "latest" versions
- Architecture detection via `uname -m` (support x86_64 and aarch64)
- Default branch is `master`
